# <IonicIcon name="create-outline" size={32} color="#ea4b71" /> Criação de Conteúdo e Templates Oficiais

Se você tem paixão por ensinar, automatizar e compartilhar, esta seção é para você. Contribuir com conteúdo para o ecossistema oficial do n8n é uma forma poderosa de ajudar a comunidade global, consolidar sua marca pessoal e ser reconhecido como um especialista.

## <IonicIcon name="apps-outline" size={24} color="#10b981" /> Tipos de Conteúdo que Você Pode Criar

### <IonicIcon name="code-working-outline" size={20} color="#0ea5e9" /> 1. Modelos (Templates) de Workflows

Uma das maneiras mais impactantes de contribuir é criando e compartilhando workflows que resolvem problemas comuns. Se você construiu uma automação útil, ela pode ser um excelente ponto de partida para milhares de pessoas.

#### O que é um bom Template?
- **Reutilizável:** Resolve um problema comum e pode ser facilmente adaptado.
- **Bem Documentado:** Use as "Sticky Notes" (notas adesivas) no editor do n8n para explicar as partes importantes.
- **Limpo e Organizado:** Nodes dispostos de forma lógica e clara.
- **Credenciais Genéricas:** Evite salvar credenciais pessoais; configure para que o usuário adicione as suas.

#### Como Submeter seu Template

### <IonicIcon name="book-outline" size={20} color="#0ea5e9" /> 2. Casos de Uso Documentados

Compartilhar um caso de uso é fundamental. Um bom exemplo prático pode inspirar e ajudar dezenas de outros usuários, além de ser a base para um excelente template ou artigo de blog.

#### Como Estruturar um Caso de Uso
1.  **Identifique o Problema:** Qual tarefa ou ineficiência este workflow resolve? Quem é o público?
2.  **Desenvolva a Solução:** Crie o workflow no n8n, organize os nodes de forma lógica e use "Sticky Notes" para documentar.
3.  **Documente:** Transforme a solução em conteúdo compartilhável. Inclua:
    *   **Título:** Claro e descritivo.
    *   **O Problema:** Cenário e dor que a automação resolve.
    *   **A Solução:** Visão geral do workflow.
    *   **Ferramentas:** Nodes e serviços utilizados.
    *   **Passo a Passo:** Detalhe as etapas críticas, configurações e manuseio de dados.
    *   **O Workflow (JSON):** Exporte e cole o código JSON em um bloco de código.

#### Compartilhe sua Criação
Com o caso de uso documentado, você pode:
- **Transformar em um Template:** Submeta ao n8n Creator Hub.
- **Escrever um Artigo:** Use como base para um tutorial no blog oficial.

### <IonicIcon name="newspaper-outline" size={20} color="#0ea5e9" /> 3. Artigos para o Blog Oficial do n8n

Escrever para o [blog oficial do n8n](https://n8n.io/blog/) é uma forma de alcançar uma audiência global. Os temas podem variar de tutoriais de workflows a artigos de opinião sobre automação.

#### Como Submeter um Artigo
1.  **Envie sua Ideia:** E-mail para `marketing@n8n.io` com o assunto `Blog contribution: [Seu Tópico]`.
2.  **Prepare o Rascunho:** Escreva o post em um Google Doc, seguindo o guia de estilo. Inclua workflows JSON em separado, se aplicável.
3.  **Créditos de Autor:** Envie um segundo Google Doc com seu nome completo, uma pequena biografia (byline) e uma foto de perfil.
4.  **Aguarde Feedback:** A equipe de marketing avaliará seu rascunho. Sem resposta em 30 dias, o post não será publicado no momento.
