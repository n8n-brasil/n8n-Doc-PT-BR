```bash title="Terminal"
npm start

# 4. Abrir no navegador
# http://localhost:3000
```

## <IonicIcon name="git-branch-outline" size={24} color="#ea4b71" /> Fluxo de Contribuição

### **Método 1: Edição Rápida (GitHub Web)**

**Ideal para:** Correções simples, typos, pequenas melhorias

1. **<IonicIcon name="search-outline" size={16} />** Navegue até a página que quer editar
2. **<IonicIcon name="create-outline" size={16} />** Clique em "Editar esta página" no final
3. **<IonicIcon name="git-branch-outline" size={16} />** GitHub criará automaticamente um fork
4. **<IonicIcon name="text-outline" size={16} />** Faça suas alterações no editor web
5. **<IonicIcon name="save-outline" size={16} />** Adicione uma descrição clara das mudanças
6. **<IonicIcon name="paper-plane-outline" size={16} />** Clique em "Propose changes"
7. **<IonicIcon name="checkmark-outline" size={16} />** Envie o Pull Request

:::info Vantagem
Você não precisa instalar nada! Tudo acontece no navegador.
:::

### **Método 2: Desenvolvimento Local**

**Ideal para:** Novos artigos, mudanças complexas, múltiplos arquivos

<Tabs>
  <TabItem value="passo1" label="1. Preparação">
    ```bash title="Terminal"
    # Criar nova branch para sua contribuição
    git checkout -b minha-contribuicao
    
    # Verificar status
    git status
    ```
    
    **Nomeação de branches:**
    - `fix/corrigir-typo-instalacao`
    - `feat/novo-tutorial-webhook`
    - `docs/atualizar-guia-docker`
  </TabItem>
  <TabItem value="passo2" label="2. Desenvolvimento">
    ```bash title="Terminal"
    # Iniciar servidor local
    npm start
    
    # Em outro terminal, fazer mudanças
    # Editar arquivos com seu editor favorito
    ```
    
    **Durante o desenvolvimento:**
    - Servidor recarrega automaticamente
    - Teste suas mudanças em tempo real
    - Verifique responsividade (mobile/desktop)
  </TabItem>
  <TabItem value="passo3" label="3. Commit e Push">
    ```bash title="Terminal"
    # Adicionar arquivos modificados
    git add .
    
    # Commit com mensagem descritiva
    git commit -m "feat: adicionar tutorial de webhook brasileiro"
    
    # Enviar para seu fork
    git push origin minha-contribuicao
    ```
    
    **Padrões de commit:**
    - `feat:` nova funcionalidade
    - `fix:` correção
    - `docs:` documentação
    - `style:` formatação
  </TabItem>
  <TabItem value="passo4" label="4. Pull Request">
    1. **Vá para seu fork no GitHub**
    2. **Clique em "Compare & pull request"**
    3. **Preencha o template:**
       - Título claro e descritivo
       - Descrição do que foi feito
       - Screenshots se aplicável
       - Checklist preenchido
    4. **Envie o PR**
    
    **Dica:** Use o template automático que aparece!
  </TabItem>
</Tabs>

## <IonicIcon name="people-outline" size={24} color="#ea4b71" /> Processo de Revisão

### **O que Acontece Após Enviar**

1. **<IonicIcon name="eye-outline" size={16} />** **Revisão Automática** (2-5 min)
   - Verificação de build
   - Testes de links
   - Validação de formatação

2. **<IonicIcon name="people-outline" size={16} />** **Revisão Humana** (1-3 dias)
   - Qualidade do conteúdo
   - Correção técnica
   - Adequação aos padrões

3. **<IonicIcon name="chatbubbles-outline" size={16} />** **Feedback e Ajustes** (conforme necessário)
   - Sugestões de melhoria
   - Correções solicitadas
   - Ajustes de estilo

4. **<IonicIcon name="checkmark-circle-outline" size={16} />** **Aprovação e Merge**
   - Contribuição aprovada
   - Código integrado ao projeto
   - Publicação automática

### **Como Responder ao Feedback**

**Se recebermos sugestões:**
- Leia com mente aberta - queremos te ajudar a melhorar
- Faça as mudanças sugeridas ou explique sua abordagem
- Pergunte se não entender algo
- Agradecemos por aceitar feedback construtivo!

**Comandos para ajustes:**
```bash title="Terminal"
# Fazer ajustes baseados no feedback
# editar arquivos...

# Commit das mudanças
git add .
git commit -m "fix: ajustes baseados no feedback da revisão"
git push origin minha-contribuicao

# O PR é atualizado automaticamente!
```

### **FAQs da Contribuição**

<details>
<summary><strong>❓ Não sei por onde começar</strong></summary>

**Resposta:** Comece com issues marcadas como `good first issue`. São problemas simples perfeitos para primeira contribuição!

**Links úteis:**
- [Issues para iniciantes](https://github.com/usuario/repo/labels/good%20first%20issue)
- [Roadmap de conteúdo](./roadmap)
- [Mentoria para novos contribuidores](./mentoria)
</details>

<details>
<summary><strong>❓ Minha ideia já existe?</strong></summary>

**Resposta:** Antes de começar, procure por:
1. Issues existentes relacionadas
2. PRs em andamento
3. Conteúdo similar na documentação

Se não encontrar nada, abra uma issue para discutir sua ideia!
</details>

<details>
<summary><strong>❓ Quanto tempo demora para aprovar?</strong></summary>

**Resposta:** 
- **Correções simples:** 1-2 dias
- **Conteúdo novo:** 3-7 dias
- **Mudanças grandes:** 1-2 semanas

Tentamos ser rápidos, mas priorizamos qualidade!
</details>

<details>
<summary><strong>❓ Posso contribuir sem saber programar?</strong></summary>

**Resposta:** **Claro!** Muitas contribuições valiosas não envolvem código:
- Melhorar textos e correções
- Adicionar exemplos e casos de uso
- Reportar problemas e sugestões
- Revisar conteúdo de outros contribuidores
- Traduzir e localizar conteúdo
</details>
