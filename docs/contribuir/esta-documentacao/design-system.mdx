# <IonicIcon name="color-palette-outline" size={32} /> Design System da Documentação n8n Brasil

> **Este documento é a fonte da verdade para a identidade visual, componentes e padrões de interface da nossa documentação.**
>
> O objetivo é criar uma experiência de usuário coesa, acessível e visualmente agradável, garantindo que todas as contribuições sigam um padrão consistente.

## Filosofia de Design

Nossa abordagem de design se baseia em três pilares:

1.  **Clareza**: A informação deve ser fácil de encontrar, ler e entender. A interface não deve ofuscar o conteúdo.
2.  **Consistência**: Componentes e padrões visuais devem ser reutilizados de forma consistente em toda a documentação para criar uma experiência previsível.
3.  **Comunidade**: O design deve refletir a energia e o espírito colaborativo da comunidade n8n no Brasil.

## <IonicIcon name="grid-outline" size={24} /> Componentes em Ação

Esta seção serve como um catálogo visual dos nossos componentes React padronizados. Use-os para construir páginas de documentação ricas e interativas.

### `<CardGrid />` e `<RepoCard />`

O `CardGrid` cria um layout responsivo para exibir cards. O `RepoCard` busca e exibe informações de um repositório do GitHub automaticamente.

**Exemplo de Uso:**
```jsx
<CardGrid>
  <RepoCard repoUrl="https://github.com/n8n-io/n8n" icon="logo-n8n" />
  <RepoCard repoUrl="https://github.com/strapi/strapi" icon="cloud-upload-outline" />
  <RepoCard repoUrl="https://github.com/payloadcms/payload" icon="server-outline" />
</CardGrid>
```

**Resultado:**

<CardGrid>
  <RepoCard repoUrl="https://github.com/n8n-io/n8n" icon="logo-n8n" />
  <RepoCard repoUrl="https://github.com/strapi/strapi" icon="cloud-upload-outline" />
  <RepoCard repoUrl="https://github.com/payloadcms/payload" icon="server-outline" />
</CardGrid>

## <IonicIcon name="color-filter-outline" size={24} /> 1. Paleta de Cores

A paleta de cores define a identidade visual do nosso projeto. Usamos um conjunto de cores primárias e secundárias para criar uma hierarquia visual clara.

### Cores Primárias
- **Principal (`--ifm-color-primary`):** `#FF4F8A` (Rosa n8n) - Usado para links, botões primários e destaques principais.
- **Secundária (`--ifm-color-secondary`):** `#1E40AF` (Azul Escuro) - Usado para títulos, subtítulos e elementos de contraste.

### Cores de Fundo
- **Light Mode (`--ifm-background-color`):** `#FFFFFF` (Branco)
- **Dark Mode (`--ifm-background-color`):** `#1E293B` (Azul Ardósia Escuro)

### Cores de Texto
- **Light Mode (`--ifm-font-color-base`):** `#374151` (Cinza Ardósia Escuro)
- **Dark Mode (`--ifm-font-color-base`):** `#E2E8F0` (Cinza Ardósia Claro)

*Esta seção será expandida com mais detalhes sobre cores de sucesso, aviso, erro, etc.*

## <IonicIcon name="text-outline" size={24} /> 2. Tipografia

A consistência tipográfica é crucial para a legibilidade.

- **Fonte Principal**: `Inter`, uma fonte sans-serif moderna e legível.
- **Fonte de Código**: `Fira Code`, uma fonte monoespaçada com ligaturas para código.

*Esta seção será expandida com tamanhos, pesos e alturas de linha para cada nível de título e parágrafo.*

## <IonicIcon name="sparkles-outline" size={24} /> 3. Iconografia

Utilizamos exclusivamente a biblioteca **Ionicons** para todos os ícones da interface, garantindo um estilo visual unificado.

- **Componente Padrão**: `<IonicIcon />`
- **Estilo Preferencial**: `outline` para a maioria dos casos.
- **Tamanhos Comuns**: 16px, 20px, 24px, 32px.

**Exemplo de Uso:**
```jsx
import IonicIcon from '@site/src/components/IonicIcon';

<IonicIcon name="rocket-outline" size={24} color="var(--ifm-color-primary)" />
```
