# <LocalIcon name="color-palette-outline" size={32} color="#ea4b71" /> Design System da Documentação n8n Brasil

> **Este documento é a fonte da verdade para a identidade visual, componentes e padrões de interface da nossa documentação.**
>
> O objetivo é criar uma experiência de usuário coesa, acessível e visualmente agradável, garantindo que todas as contribuições sigam um padrão consistente.

---

## <LocalIcon name="bulb-outline" size={24} color="#ea4b71" /> Filosofia de Design

Nossa abordagem de design se baseia em três pilares:

1.  **Clareza**: A informação deve ser fácil de encontrar, ler e entender. A interface não deve ofuscar o conteúdo.
2.  **Consistência**: Componentes e padrões visuais devem ser reutilizados de forma consistente em toda a documentação para criar uma experiência previsível.
3.  **Comunidade**: O design deve refletir a energia e o espírito colaborativo da comunidade n8n no Brasil.

## <LocalIcon name="color-filter-outline" size={24} color="#ea4b71" /> Identidade Visual

### <LocalIcon name="color-palette-outline" size={20} color="#ea4b71" /> Paleta de Cores

A paleta de cores define a identidade visual do nosso projeto. Usamos um conjunto de cores primárias e secundárias para criar uma hierarquia visual clara.

#### Cores Primárias
- **Principal (`--ifm-color-primary`):** `#FF4F8A` (Rosa n8n) - Usado para links, botões primários e destaques principais.
- **Secundária (`--ifm-color-secondary`):** `#1E40AF` (Azul Escuro) - Usado para títulos, subtítulos e elementos de contraste.

#### Cores de Fundo
- **Light Mode (`--ifm-background-color`):** `#FFFFFF` (Branco)
- **Dark Mode (`--ifm-background-color`):** `#1E293B` (Azul Ardósia Escuro)

#### Cores de Texto
- **Light Mode (`--ifm-font-color-base`):** `#374151` (Cinza Ardósia Escuro)
- **Dark Mode (`--ifm-font-color-base`):** `#E2E8F0` (Cinza Ardósia Claro)

*Esta seção será expandida com mais detalhes sobre cores de sucesso, aviso, erro, etc.*

### <LocalIcon name="text-outline" size={20} color="#ea4b71" /> Tipografia

A consistência tipográfica é crucial para a legibilidade.

- **Fonte Principal**: `Inter`, uma fonte sans-serif moderna e legível.
- **Fonte de Código**: `Fira Code`, uma fonte monoespaçada com ligaturas para código.

*Esta seção será expandida com tamanhos, pesos e alturas de linha para cada nível de título e parágrafo.*

### <LocalIcon name="sparkles-outline" size={20} color="#ea4b71" /> Iconografia

Utilizamos exclusivamente a biblioteca **Ionicons** para todos os ícones da interface, garantindo um estilo visual unificado.

- **Componente Padrão**: `<LocalIcon />`
- **Estilo Preferencial**: `outline` para a maioria dos casos.
- **Tamanhos Comuns**: 16px, 20px, 24px, 32px.

**Exemplo de Uso:**
```jsx

<LocalIcon name="rocket-outline" size={24} color="var(--ifm-color-primary)" />
```

## <LocalIcon name="construct-outline" size={24} color="#ea4b71" /> Padrões de Uso

### <LocalIcon name="sparkles-outline" size={20} color="#ea4b71" /> Guia de Uso de Ícones Ionicons

> **Padrão obrigatório para toda a documentação e componentes React.**

#### Tamanhos Permitidos

| Uso                  | Tamanho (px) |
|----------------------|--------------|
| Ícones em listas, stats, detalhes | 16           |
| Ícones em botões, cards, navegação| 20           |
| Ícones em títulos, seções, destaques | 24, 32      |
| Ilustrações, banners, hero         | 48           |

> **Evite usar 64px, exceto para ilustrações muito específicas.**

#### Estilo
- **Apenas ícones outline** (`-outline`).
- **Exceção:** ícones de logo (ex: `logo-github`).
- Não usar variantes `-filled`, `-sharp`, etc.

#### Cor
- **Nunca** use cor hardcoded (ex: `color="#FF0000"`).
- Sempre use `currentColor` (padrão) ou variável CSS, ex: `color="var(--ifm-color-primary)"`.
- O ícone herda a cor do contexto (texto, botão, etc).

#### Acessibilidade
- O componente já define `aria-hidden="true"` por padrão.

#### Exemplo de Uso Correto
```jsx
<LocalIcon name="rocket-outline" size={24} color="var(--ifm-color-primary)" />
<LocalIcon name="star-outline" size={16} />
<LocalIcon name="logo-github" size={20} />
```

#### Exemplo de Uso Incorreto
```jsx
// NÃO FAÇA:
<LocalIcon name="star" size={18} color="#FF0000" /> // tamanho e cor fora do padrão
<LocalIcon name="star-filled" size={24} /> // estilo não permitido
```

#### Boas Práticas
- Sempre consulte esta tabela antes de adicionar um novo ícone.
- Se precisar de um tamanho diferente, discuta antes com a equipe.
- Prefira reutilizar ícones já usados em outros lugares.

## <LocalIcon name="code-outline" size={24} color="#ea4b71" /> Estrutura CSS e Convenções

A partir de 2024, a documentação adota uma estrutura CSS modularizada para facilitar manutenção, escalabilidade e performance. Veja como contribuir corretamente:

### <LocalIcon name="folder-outline" size={20} color="#ea4b71" /> Estrutura de Arquivos

```
src/css/
├── variables.css      # Variáveis globais (cores, tipografia, espaçamentos)
├── base.css           # Reset, tipografia e estilos base
├── components.css     # Estilos globais de componentes (BEM)
├── utilities.css      # Classes utilitárias reutilizáveis
└── custom.css         # Importa todos os arquivos acima
```

### <LocalIcon name="settings-outline" size={20} color="#ea4b71" /> Convenções

- **BEM**: Use o padrão `bloco__elemento--modificador` para classes de componentes globais.
- **Variáveis globais**: Sempre utilize as variáveis de `variables.css` para cores, fontes, espaçamentos, etc.
- **Utilitários**: Prefira classes utilitárias para tipografia, espaçamento, peso, etc, sempre que possível.
- **Componentes React**: Os módulos CSS dos componentes devem consumir variáveis globais e utilitários, evitando duplicação.

### <LocalIcon name="code-slash-outline" size={20} color="#ea4b71" /> Exemplos

```css
/* BEM */
.card-details__separator { margin: 0 0.25rem; }

/* Utilitário */
.text-xs { font-size: var(--font-size-xs) !important; }

/* Variável global */
color: var(--ifm-color-primary);
```

### <LocalIcon name="checkmark-circle-outline" size={20} color="#ea4b71" /> Boas Práticas

- Não duplique regras já existentes em utilitários ou variáveis.
- Documente novas classes utilitárias ou variáveis criadas.
- Prefira sempre a reutilização.
