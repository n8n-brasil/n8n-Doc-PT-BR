# Fluxo de Dados Dentro dos Nodes

O n8n processa dados como uma série de itens. Cada item é um objeto que contém os dados que você está trabalhando. Entender como esses itens fluem através dos nodes é fundamental para criar workflows eficientes.

<IonicIcon name="chevron-forward-outline" style={{fontSize: '24px', color: '#ea4b71'}} />

## Exemplo de Dados

```json\n[\n {"json": {"nome": "João", "email": "joao@email.com"}},\n {"json": {"nome": "Maria", "email": "maria@email.com"}},\n {"json": {"nome": "Pedro", "email": "pedro@email.com"}}\n]\n```\n```\n\nSe você conectar um node de email a este, ele enviará **3 emails** - um para cada cliente.\n\n<IonicIcon name="settings-outline" style={{fontSize: '24px', color: '#ea4b71'}} />\n\n## Configurações de Execução\n\n### Execute Once\n\nAlguns nodes têm a opção **Execute Once** nas configurações. Quando ativada:\n\n- O node processa apenas o **primeiro item** recebido\n- Ignora todos os outros itens\n- Útil para operações que devem acontecer apenas uma vez\n\n### Exemplo de Uso\n\n```mermaid\ngraph LR\n    A[Webhook] --> B[HTTP Request]\n    B --> C[Send Email - Execute Once]\n\n    A --> A1[1 item]\n    B --> B1[5 items]\n    C --> C1[1 email enviado]\n```\n\n```
