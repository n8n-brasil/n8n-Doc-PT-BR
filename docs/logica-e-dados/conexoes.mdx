#ionicicon-namegit-network-outline-style-fontsize-24px color: '#cor-ea4b71-conexes-entre Nodes

As conexões são a base de qualquer workflow no n8n, definindo como os dados fluem de um node para outro e criando a lógica de execução da sua automação.

#ionicicon-nameinformation-circle-outline-style-fontsize-24px color: '#cor-ea4b71-o-que são Conexões?

Conexões são linhas que conectam nodes, definindo:

- **Ordem de execução** dos nodes
- **Fluxo de dados** entre operações
- **Lógica condicional** do workflow
- **Tratamento de erros** e exceções

#tipos-de-conexes

#main-connection-conexo-principal-a conexão padrão que transporta dados de sucesso:

- **Cor**: Azul (padrão)
- **Função**: Passa dados de um node para outro
- **Uso**: Fluxo normal de execução

#error-connection-conexo-de-erro

Trata falhas e exceções:

- **Cor**: Vermelha
- **Função**: Captura erros e falhas
- **Uso**: Tratamento de exceções

#optional-connection-conexo-opcional-conexes que podem ou não ser executadas:

- **Cor**: Cinza
- **Função**: Fluxos condicionais
- **Uso**: Lógica alternativa

#ionicicon-namechevron-forward-outline-style-fontsize-24px color: '#cor-ea4b71-como-criar Conexões

#mtodo-visual-1-clique-no ponto de saída de um node
2. **Arraste** até o ponto de entrada de outro node
3. **Solte** para criar a conexão

#mtodo-rpido-1-selecione-o node de origem
2. **Pressione** `Ctrl/Cmd + K`
3. **Digite** o nome do node de destino
4. **Pressione** `Enter`

#mltiplas-conexes-um-node-pode ter múltiplas conexões de saída:

- **Conexões paralelas**: Execução simultânea
- **Conexões condicionais**: Baseadas em condições
- **Conexões de erro**: Para tratamento de falhas

#ionicicon-namecode-slash-outline-style-fontsize-24px color: '#cor-ea4b71-tipos-de Fluxo

#fluxo-linear-a-forma-mais simples de conexão:

```
Trigger → Node A → Node B → Node C
```

**Características:**
- Execução sequencial
- Dados passam de um para outro
- Falha em qualquer node para o workflow

#fluxo-paralelo-mltiplas-conexes-saindo do mesmo node:

```
Trigger → Node A → Node B
        → Node C
        → Node D
```

**Características:**
- Execução simultânea
- Cada ramo processa independentemente
- Útil para operações independentes

#fluxo-condicional-conexes-baseadas-em condições:

```
Trigger → IF Node → Node A (se verdadeiro)
                → Node B (se falso)
```

**Características:**
- Decisões baseadas em dados
- Fluxos alternativos
- Lógica de negócio complexa

#fluxo-de-erro-tratamento-de falhas e exceções:

```
Trigger → Node A → Node B (sucesso)
        → Error Handler (falha)
```

**Características:**
- Captura erros automaticamente
- Permite recuperação
- Logging de falhas

#ionicicon-namesettings-outline-style-fontsize-24px color: '#cor-ea4b71-configuraes-de Conexão

#condies-de-conexo-defina-quando uma conexão deve ser executada:

```javascript
// Executar apenas se status for 'ativo'
{{ $json.status === 'ativo' }}

// Executar se valor for maior que 100
{{ $json.valor > 100 }}

// Executar se email existir
{{ $json.email && $json.email.includes('@') }}
```

#mltiplas-condies-combine-condies-com operadores lógicos:

```javascript
// E lógico (AND)
{{ $json.status === 'ativo' && $json.valor > 100 }}

// OU lógico (OR)
{{ $json.prioridade === 'alta' || $json.urgente === true }}

// Negação (NOT)
{{ !$json.processado }}
```

#expresses-complexas-javascript-verificar múltiplos campos
{{ $json.nome && $json.email && $json.telefone }}

// Validação de dados
{{ $json.idade >= 18 && $json.idade <= 65 }}

// Verificar tipo de dados
{{ typeof $json.valor === 'number' && $json.valor > 0 }}
```

#ionicicon-namewarning-outline-style-fontsize-24px color: '#cor-ea4b71-problemas-comuns

#conexes-circulares-problema-workflow-entra em loop infinito

**Solução:**
- Evite conectar um node de volta a si mesmo
- Use condições para quebrar loops
- Implemente contadores de iteração

#conexes-perdidas-problema-dados-no chegam ao destino

**Solução:**
- Verifique se a conexão está ativa
- Confirme condições de conexão
- Teste com dados de exemplo

#performance-problema-workflow-lento-ou travado

**Solução:**
- Limite conexões paralelas excessivas
- Use filtros para reduzir volume de dados
- Implemente timeouts adequados

#ionicicon-nameanalytics-outline-style-fontsize-24px color: '#cor-ea4b71-debugging-de Conexões

#verificar-fluxo-1-execute-step by step** para ver o caminho dos dados
2. **Verifique dados** em cada node
3. **Analise logs** de execução
4. **Teste condições** individualmente

#ferramentas-de-debug-execution Log**: Histórico completo
- **Data Viewer**: Visualizar dados em cada step
- **Connection Tester**: Validar conexões

#logs-de-conexo-javascript Log no início de cada node

// Log de dados importantes

```
