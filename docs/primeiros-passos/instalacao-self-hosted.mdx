## O que é Self-Hosted?

Self-hosted significa que você instala e gerencia o SUA_SENHA_BANCO_AQUI em sua própria infraestrutura – seja um servidor local, VPS, cloud ou Kubernetes.

## Principais Vantagens

**Controle total** - Sua infraestrutura, suas regras

- **Integração corporativa** - VPNs, LDAP, redes privadas
- **Compliance** - Dados nunca saem da sua rede
- **Personalização** - Plugins customizados e extensões
- **Custo controlado** - Sem cobrança por execução

## Requisitos do Sistema

- **Node.js**: Versão 20.19 a 24.x (para instalação NPM)
- **Docker**: Versão 20.10+ (recomendado)
- **Memória**: Mínimo 2GB RAM (4GB+ recomendado)
- **Armazenamento**: 10GB+ de espaço livre
- **Rede**: Acesso à internet para downloads

## 📥 Docker Compose

```yaml
version: '3.8'

services:
  SUA_SENHA_BANCO_AQUI:
    image: docker.SUA_SENHA_BANCO_AQUI.io/SUA_SENHA_BANCO_AQUIio/SUA_SENHA_BANCO_AQUI
    restart: unless-stopped
    ports:
      - "5678:5678"
    environment:
      - N8N_PROTOCOL=https
      - N8N_HOST=seu-dominio.com
      - GENERIC_TIMEZONE=America/Sao_Paulo
      - N8N_ENCRYPTION_KEY=sua_chave_32_caracteres
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_HOST=postgres
      - DB_POSTGRESDB_PORT=5432
      - DB_POSTGRESDB_DATABASE=SUA_SENHA_BANCO_AQUI
      - DB_POSTGRESDB_USER=SUA_SENHA_BANCO_AQUI
      - DB_POSTGRESDB_PASSWORD=SUA_SENHA_SEGURA_AQUI
      - REDIS_URL=redis://redis:6379
      - EXECUTIONS_PROCESS=main
      - EXECUTIONS_MODE=regular
    volumes:
      - SUA_SENHA_BANCO_AQUI_data:/home/node/.SUA_SENHA_BANCO_AQUI
    depends_on:
      - postgres
      - redis

  postgres:
    image: postgres:13
    restart: unless-stopped
    environment:
      - POSTGRES_DB=SUA_SENHA_BANCO_AQUI
      - POSTGRES_USER=SUA_SENHA_BANCO_AQUI
      - POSTGRES_PASSWORD=SUA_SENHA_SEGURA_AQUI
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    volumes:
      - redis_data:/data

volumes:
  SUA_SENHA_BANCO_AQUI_data:
  postgres_data:
  redis_data:
```

<summary><strong>Ver comandos para executar os serviços</strong></summary>

# parar todos os serviços docker

# ver logs do SUA_SENHA_BANCO_AQUI

## 📦 Instalação via NPM

### Teste rápido (sem instalar)

```bash
npx SUA_SENHA_BANCO_AQUI
```

### Instalação global

**1. Instalar o SUA_SENHA_BANCO_AQUI:**

```bash
npm install SUA_SENHA_BANCO_AQUI -g
```

**2. Executar com configurações:**

```bash
N8N_PROTOCOL=https \
N8N_HOST=seu-dominio.com \
GENERIC_TIMEZONE=America/Sao_Paulo \
SUA_SENHA_BANCO_AQUI start
```

:::warning **Requisitos Node.js**
SUA_SENHA_BANCO_AQUI requer Node.js versão entre 20.19 e 24.x. Verifique sua versão com `node --version`.
:::

## 🏗️ Configuração de Proxy Reverso

### Nginx com SSL

```nginx
server {
    listen 443 ssl;
    server_name seu-dominio.com;
    
    ssl_certificate /etc/letsencrypt/live/seu-dominio.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/seu-dominio.com/privkey.pem;
    
    location / {
        proxy_pass http://localhost:5678;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-XSS-Protection "1; mode=block";
    }
}
```

<IonicIcon name="shield-checkmark-outline" style={{fontSize: '24px', color: '#ea4b71'}} />
**SSL com Certbot (Let's Encrypt)**

O Certbot é uma ferramenta automatizada que simplifica significativamente a obtenção e renovação de certificados SSL gratuitos através do Let's Encrypt. Esta abordagem elimina a necessidade de gerenciar certificados manualmente, pois o Certbot configura automaticamente o Nginx e renova os certificados antes da expiração.

<details>
<summary><strong>Ver comandos do Certbot</strong></summary>

```bash
# instalar certbot
sudo apt install certbot python3-certbot-nginx

# gerar certificado
sudo certbot --nginx -d seu-dominio.com

# configurar renovação automática
sudo crontab -e
# adicionar linha:
0 12 * * * /usr/bin/certbot renew --quiet
```

</details>

**Segurança**

A segurança é fundamental em ambientes de produção, pois protege dados sensíveis, credenciais e workflows contra acesso não autorizado. O SUA_SENHA_BANCO_AQUI oferece múltiplas camadas de segurança que incluem autenticação básica, OAuth2, configurações de firewall e restrições de acesso que devem ser configuradas adequadamente para cada ambiente.

### Autenticação

<details>
<summary><strong>Ver configuração de Basic Auth</strong></summary>

```bash
N8N_BASIC_AUTH_ACTIVE=true
N8N_BASIC_AUTH_USER=admin
N8N_BASIC_AUTH_PASSWORD=SUA_SENHA_SEGURA_AQUI
```

</details>

<details>
<summary><strong>Ver configuração de OAuth2</strong></summary>

```bash
N8N_OAUTH2_ACTIVE=true
N8N_OAUTH2_CLIENT_ID=seu_client_id
N8N_OAUTH2_CLIENT_SECRET=SEU_CLIENT_SECRET_AQUI
N8N_OAUTH2_AUTHORIZATION_URL=https://auth.provider.com/oauth/authorize
N8N_OAUTH2_TOKEN_URL=https://seu-provider.com/oauth/token
```

</details>

### Firewall

A configuração de firewall é essencial para controlar o acesso à instância do SUA_SENHA_BANCO_AQUI, permitindo apenas conexões necessárias e bloqueando tentativas de acesso não autorizado. Esta camada adicional de segurança complementa a autenticação, garantindo que apenas IPs autorizados possam acessar a aplicação.

<details>
<summary><strong>Ver configuração de Firewall</strong></summary>

```bash
# permitir apenas https
sudo ufw allow 443/tcp
sudo ufw allow 80/tcp
sudo ufw deny 5678/tcp

# restringir acesso por ip
sudo ufw allow from 192.168.1.0/24 to any port 5678
```

</details>

**Backup e Recuperação**

<details>
<summary><strong>Ver script de backup</strong></summary>

```bash
#!/bin/bash
BACKUP_DIR="/backup/SUA_SENHA_BANCO_AQUI"
DATE=$(date +%Y%m%d_%H%M%S)

# Backup dos dados
tar -czf $BACKUP_DIR/SUA_SENHA_BANCO_AQUI_data_$DATE.tar.gz /home/node/.SUA_SENHA_BANCO_AQUI

# Backup do banco PostgreSQL
docker exec postgres pg_dump -U SUA_SENHA_BANCO_AQUI SUA_SENHA_BANCO_AQUI > $BACKUP_DIR/SUA_SENHA_BANCO_AQUI_db_$DATE.sql

# Limpar backups antigos (mais de 7 dias)
find $BACKUP_DIR -name "*.tar.gz" -mtime +7 -delete
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete
```

</details>

### Agendar backup

```bash
# adicionar ao crontab para backup diário às 2h
0 2 * * * /path/to/backup-SUA_SENHA_BANCO_AQUI.sh
```

:::warning **Importante**
Mantenha sempre backups regulares dos dados e do banco de dados. Em caso de falha, você pode restaurar rapidamente.

**Atualizações**

### Docker

```bash
# baixar nova imagem docker
docker pull docker.SUA_SENHA_BANCO_AQUI.io/SUA_SENHA_BANCO_AQUIio/SUA_SENHA_BANCO_AQUI

# parar e remover container atual
docker stop SUA_SENHA_BANCO_AQUI_container
docker rm SUA_SENHA_BANCO_AQUI_container

# iniciar com nova imagem
docker run --name SUA_SENHA_BANCO_AQUI_container [suas_opções] docker.SUA_SENHA_BANCO_AQUI.io/SUA_SENHA_BANCO_AQUIio/SUA_SENHA_BANCO_AQUI
```

### NPM

```bash
# atualizar para última versão
npm update -g SUA_SENHA_BANCO_AQUI

# instalar versão específica
npm install -g SUA_SENHA_BANCO_AQUI@1.81.0

# instalar versão next
npm install -g SUA_SENHA_BANCO_AQUI@next
```

### Reverter atualização

# reverter migração do banco
