Encontre respostas rápidas para as perguntas mais comuns sobre o n8n. Se sua dúvida não estiver aqui, consulte nossa [comunidade](../comunidade/index.mdx) ou [GitHub](https://github.com/tatyquebralayout/n8n-Doc-pt-BR).

#ionicicon-namerocket-outline-style-fontsize-24px color: '#cor-ea4b71-primeiros-passos

#o-que-o-n8n

Para uma explicação completa, **[consulte nossa seção de Conceitos Fundamentais](./conceitos-fundamentais)**. Em resumo, o n8n é uma plataforma de automação de workflow open-source que permite conectar diferentes aplicações e serviços através de uma interface visual.

#o-n8n-gratuito-sim O n8n é totalmente gratuito e open-source. Você pode usar todos os recursos sem custo, incluindo:

- Workflows ilimitados
- Execuções ilimitadas
- Todos os nodes disponíveis
- Self-hosting completo

#preciso-de-conhecimento-tcnico-para usar o n8n?

Não necessariamente. O n8n foi projetado para ser intuitivo com sua interface visual. Você pode começar criando workflows simples arrastando e soltando nodes, mesmo sem conhecimento de programação.

#qual-a-diferena-entre-n8n Cloud e Self-hosted?

- **n8n Cloud**: Serviço gerenciado pela equipe n8n, sem necessidade de configuração de servidor
- **Self-hosted**: Você instala e gerencia o n8n em seu próprio servidor, com controle total sobre dados e configuração

#ionicicon-namesettings-outline-style-fontsize-24px color: '#cor-ea4b71-instalao-e Configuração

#qual-mtodo-de-instalao recomendado?

Para iniciantes, recomendamos:

1. **n8n Cloud** - Mais simples, sem configuração
2. **Docker** - Para self-hosting, mais fácil de gerenciar
3. **npm** - Para desenvolvedores que preferem controle total

#posso-instalar-o-n8n-no Windows?

Sim! O n8n funciona perfeitamente no Windows. Você pode usar:

- Docker Desktop
- WSL2 (Windows Subsystem for Linux)
- Instalação direta via npm

#quais-so-os-requisitos-mnimos

- **CPU**: 1 core
- **RAM**: 1GB (recomendado 2GB+)
- **Armazenamento**: 1GB livre
- **Node.js**: Versão 16 ou superior

#como-fao-backup-dos-meus workflows?

Os workflows são salvos automaticamente. Para backup manual:

- Exporte workflows individuais via interface
- Use a API para backup completo
- Faça backup do banco de dados (se self-hosted)

#ionicicon-namegit-branch-outline-style-fontsize-24px color: '#cor-ea4b71-workflows

#quantos-workflows-posso-criar-no há limite! Você pode criar quantos workflows quiser, tanto no n8n Cloud quanto self-hosted.

#posso-agendar-workflows-para-execuo automática?

Sim! Use o **Schedule Trigger** para executar workflows em intervalos específicos:

- A cada minuto, hora, dia
- Em horários específicos
- Com expressões cron personalizadas

#como-fao-para-que-workflows falhem graciosamente?

Use o **Error Trigger** para capturar erros e:

- Enviar notificações
- Tentar novamente automaticamente
- Executar workflows de recuperação
- Registrar logs de erro

#posso-compartilhar-workflows-com-outros usuários?

Sim! Você pode:

- Exportar workflows como JSON
- Compartilhar via URL (n8n Cloud)
- Usar controle de versão (Git)
- Publicar na comunidade n8n

#ionicicon-nameshield-checkmark-outline-style-fontsize-24px color: '#cor-ea4b71-segurana-e Credenciais

#como-o-n8n-protege-minhas credenciais?

- Credenciais são criptografadas no banco de dados
- Senhas nunca são exibidas em texto plano
- Suporte a variáveis de ambiente
- Integração com gerenciadores de segredos

#posso-usar-o-n8n-com dados sensíveis?

Sim, mas recomendamos:

- Usar self-hosting para dados críticos
- Implementar autenticação forte
- Usar HTTPS em produção
- Configurar firewalls adequados

#o-n8n-suporta-2fa-sim O n8n suporta autenticação de dois fatores para maior segurança.

#como-gerencio-permisses-de-usurios

O n8n oferece:

- Roles personalizáveis
- Permissões granulares por workflow
- Controle de acesso por credenciais
- Logs de auditoria

#ionicicon-namecode-outline-style-fontsize-24px color: '#cor-ea4b71-nodes-e Integrações

#quantos-nodes-esto-disponveis-o n8n possui centenas de nodes, incluindo:

- **Built-in nodes**: HTTP, Code, Logic, etc.
- **App nodes**: Integrações com serviços populares
- **Community nodes**: Criados pela comunidade
- **Custom nodes**: Que você pode criar

#posso-criar-meus-prprios-nodes

Sim! Você pode:

- Criar nodes customizados em JavaScript/TypeScript
- Publicar na comunidade n8n
- Usar nodes privados em sua instalação
- Contribuir para o projeto oficial

#o-n8n-suporta-apis-brasileiras

Sim! O n8n pode integrar com qualquer API, incluindo:

- APIs governamentais (Receita Federal, etc.)
- Serviços financeiros brasileiros
- APIs de e-commerce locais
- Serviços de pagamento brasileiros

#como-fao-para-integrar-com sistemas legados?

Use os nodes:

- **HTTP Request** para APIs REST
- **Webhook** para receber dados
- **Code** para lógica customizada
- **Database** para sistemas legados

#ionicicon-namespeedometer-outline-style-fontsize-24px color: '#cor-ea4b71-performance-e Escalabilidade

#quantas-execues-simultneas-o-n8n suporta?

Depende da sua configuração:

- **n8n Cloud**: Até 100 execuções simultâneas
- **Self-hosted**: Limitado apenas pelos recursos do servidor
- **Clustering**: Para alta disponibilidade

#como-otimizo-a-performance-dos workflows?

- Use **Split In Batches** para grandes volumes
- Implemente **caching** quando possível
- Evite loops desnecessários
- Use **queues** para processamento assíncrono

#posso-escalar-horizontalmente-sim-o n8n suporta:

- Múltiplas instâncias
- Load balancing
- Clustering com Redis
- Execuções distribuídas

#como-monitoro-a-performance-use

- **Execution Logs** para análise detalhada
- **Metrics** para monitoramento
- **Alertas** para problemas
- **Dashboard** para visão geral
