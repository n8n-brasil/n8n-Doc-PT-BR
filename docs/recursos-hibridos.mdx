# Recursos Híbridos de Documentação

## <IonicIcon name="flask-outline" style={{ fontSize: '24px', color: '#ea4b71' }} /> Visão Geral

Esta página demonstra como podemos integrar funcionalidades avançadas do **reStructuredText** e **Sphinx** mantendo o **Docusaurus** como tema principal. Esta abordagem híbrida oferece o melhor dos três mundos: simplicidade do Markdown, poder do RST e funcionalidades especializadas do Sphinx.

:::info Funcionalidade Híbrida
Esta página usa componentes React customizados que processam sintaxe reStructuredText e Sphinx, mantendo a compatibilidade com o Docusaurus.
:::

.. n8n-workflow:: lead-processing-workflow
   :title: Workflow de Processamento de Leads
   :description: Automatiza o processamento completo de leads do formulário web até o CRM
   :nodes: 8
   :executions: 2500/month
   :complexity: intermediate
   
   .. workflow-step:: webhook-trigger
      :description: Recebe dados do formulário de contato
      :config: { "method": "POST", "path": "/leads", "responseMode": "responseNode" }
      :validation: |
         - Email obrigatório
         - Nome obrigatório
         - Telefone opcional
   
   .. workflow-step:: data-validation
      :description: Valida e limpa dados de entrada
      :code: |
         const email = $input.item.json.email;
         const phone = $input.item.json.phone;
         const name = $input.item.json.name;
         
         // Validações
         if (!email || !email.includes('@')) {
           throw new Error('Email inválido');
         }
         
         if (!name || name.length < 2) {
           throw new Error('Nome inválido');
         }
         
         return {
           ...$input.item.json,
           validated: true,
           timestamp: new Date().toISOString()
         };
   
   .. workflow-step:: lead-scoring
      :description: Calcula score do lead baseado em múltiplos critérios
      :code: |
         const data = $input.item.json;
         let score = 0;
         
         // Score por tipo de empresa
         if (data.companyType === 'enterprise') score += 50;
         else if (data.companyType === 'startup') score += 30;
         else score += 10;
         
         // Score por valor estimado
         if (data.estimatedValue > 100000) score += 40;
         else if (data.estimatedValue > 50000) score += 25;
         else score += 10;
         
         // Score por localização
         if (['SP', 'RJ', 'MG'].includes(data.state)) score += 20;
         
         return {
           ...data,
           leadScore: score,
           priority: score > 80 ? 'high' : score > 50 ? 'medium' : 'low'
         };
   
   .. workflow-step:: crm-update
      :description: Atualiza CRM com dados do lead
      :service: salesforce
      :config: { "operation": "create", "resource": "lead", "additionalFields": { "LeadSource": "Website" } }
      :error-handling: |
         - Retry automático em caso de falha
         - Log de erro para análise posterior
         - Notificação para equipe técnica
   
   .. workflow-step:: slack-notification
      :description: Notifica equipe de vendas sobre novo lead
      :channel: "#leads"
      :template: |
         🎯 *Novo Lead Qualificado*
         *Nome:* {name}
         *Email:* {email}
         *Empresa:* {company}
         *Score:* {leadScore}/100
         *Prioridade:* {priority}
         *Valor Estimado:* R$ {estimatedValue}
   
   .. workflow-step:: email-followup
      :description: Envia email de boas-vindas personalizado
      :template: "welcome-email"
      :delay: "5 minutes"
      :personalization: |
         - Nome do lead
         - Tipo de empresa
         - Score calculado
         - Próximos passos específicos
   
   .. workflow-step:: analytics-tracking
      :description: Registra evento no Google Analytics
      :service: google-analytics
      :config: { "eventName": "lead_conversion", "eventCategory": "marketing" }
   
   .. workflow-step:: database-backup
      :description: Salva backup dos dados em banco local
      :service: postgresql
      :config: { "operation": "insert", "table": "leads_backup" }
`} />

### Documentação de API Estruturada

<HybridProcessor content={`
.. http:post:: /api/v1/workflows/{id}/execute
   :synopsis: Executa um workflow específico do n8n
   
   Executa um workflow n8n pelo ID fornecido. Esta API permite executar workflows
   programaticamente, útil para integrações externas e automações de nível superior.
   
   :param id: ID único do workflow
   :type id: string
   :param data: Dados de entrada para o workflow
   :type data: object
   :param options: Opções de execução (opcional)
   :type options: object
   :param waitForCompletion: Aguardar conclusão da execução (padrão: false)
   :type waitForCompletion: boolean
   
   :>json object success: Status da execução (true/false)
   :>json string executionId: ID único da execução
   :>json object result: Resultado do workflow (apenas se waitForCompletion=true)
   :>json string status: Status da execução (running/completed/failed)
   :>json number startedAt: Timestamp de início da execução
   :>json number finishedAt: Timestamp de conclusão (apenas se concluído)
   
   :status 200: Workflow executado com sucesso
   :status 400: Dados de entrada inválidos
   :status 404: Workflow não encontrado
   :status 500: Erro interno do servidor
   
   **Exemplo de requisição:**
   
   .. sourcecode:: bash
      
      curl -X POST "https://api.n8n.io/v1/workflows/abc123/execute" \\
           -H "Content-Type: application/json" \\
           -H "Authorization: Bearer YOUR_API_KEY" \\
           -d '{
             "data": {
               "email": "test@example.com",
               "name": "João Silva",
               "company": "Empresa ABC",
               "estimatedValue": 50000
             },
             "options": {
               "waitForCompletion": true
             }
           }'
   
   **Exemplo de resposta (waitForCompletion=false):**
   
   .. sourcecode:: json
      
      {
        "success": true,
        "executionId": "exec_789",
        "status": "running",
        "startedAt": 1640995200000
      }
   
   **Exemplo de resposta (waitForCompletion=true):**
   
   .. sourcecode:: json
      
      {
        "success": true,
        "executionId": "exec_789",
        "status": "completed",
        "startedAt": 1640995200000,
        "finishedAt": 1640995260000,
        "result": {
          "processed": true,
          "crmUpdated": true,
          "notificationSent": true,
          "leadScore": 85,
          "priority": "high"
        }
      }
   
   **Códigos de erro:**
   
   .. sourcecode:: json
      
      {
        "success": false,
        "error": {
          "code": "WORKFLOW_NOT_FOUND",
          "message": "Workflow with ID 'abc123' not found",
          "details": {
            "workflowId": "abc123",
            "availableWorkflows": ["def456", "ghi789"]
          }
        }
      }
`} />
