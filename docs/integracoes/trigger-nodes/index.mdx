Introdução aos Trigger Nodes

Os trigger nodes são o ponto de partida de qualquer workflow automatizado. Eles permitem que seus workflows sejam iniciados automaticamente baseados em eventos, horários ou ações específicas.

## Tipos de Trigger Nodes

### <IonicIcon name="play-circle-outline" style={{fontSize: '16px', color: '#ea4b71'}} /> Manual Trigger

Permite executar workflows manualmente, ideal para testes e desenvolvimento.

### <IonicIcon name="time-outline" style={{fontSize: '16px', color: '#ea4b71'}} /> Schedule Trigger

Executa workflows automaticamente em horários específicos usando expressões cron.

### <IonicIcon name="git-network-outline" style={{fontSize: '16px', color: '#ea4b71'}} /> Webhook Trigger

Recebe dados de sistemas externos através de endpoints HTTP.

### <IonicIcon name="document-outline" style={{fontSize: '16px', color: '#ea4b71'}} /> App Triggers

Triggers específicos de aplicativos para automações baseadas em eventos.
